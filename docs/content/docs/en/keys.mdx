## Trackball modes

The LED color always shows which mode is active.

| Mode | The ballтАж | Turn on/off | LED |
|------|-----------|-------------|-----|
| **Pointer** | moves the mouse pointer, accelerating with speed | always on | default |
| **Sniping** | moves the pointer slowly and precisely, no acceleration | hold `SNIPING` ┬╖ toggle `SNP_TOG` ┬╖ auto while `PT_Z`/`PT_SLSH` held | ЁЯФ┤ red |
| **Drag-scroll** | scrolls, sticking to one axis; a flick coasts with inertia | hold `DRGSCRL` ┬╖ toggle `DRG_TOG` ┬╖ tap `DRGT_CRTT` | ЁЯЯв green |
| **Carret** | moves the text cursor with arrow keys; jumps word-wise when sniping is on | hold `CARRETM` ┬╖ toggle `CRT_TOG` ┬╖ hold `DRGT_CRTT` | ЁЯФ╡ azure |

Drag-scroll and carret are mutually exclusive тАФ turning one on turns the
other off.

## Pointer layer

Held via `PT_Z` (Z) or `PT_SLSH` (/). The 3x5 core, identical on every board:

```text
QK_BOOT  DPI_RMOD DPI_MOD  S_D_RMOD S_D_MOD  тФВ  S_D_MOD  S_D_RMOD DPI_MOD  DPI_RMOD QK_BOOT
LGUI     LALT     LCTL     LSFT     DRGT_CRTTтФВ  CRT_TOG  RSFT     RCTL     RALT     RGUI
 ┬╖ ┬╖     DRGSCRL  SNIPING  DRG_TOG  SNP_TOG  тФВ  SNP_TOG  DRG_TOG  SNIPING  DRGSCRL   ┬╖ ┬╖
                  BTN2     BTN1     BTN3     тФВ  BTN_LOCK BTN1
```

`DPI_MOD`/`DPI_RMOD` step the pointer DPI up/down, `S_D_MOD`/`S_D_RMOD` do
the same for sniping DPI.

## Custom keycodes

Declared in `users/wh1teend/common.h`, handled in `common.c`. The web
configurator shows them as unknown codes; assign via the **Any** key using
the hex value.

<Callout type="warn" title="Browser edits don't survive a flash">
Layers re-seed from `keymap.c` on every new build. Argos combos, tap dances
and DPI settings live elsewhere in EEPROM and survive.
</Callout>

| Key | Hex (Any) | Behavior |
|-----|-----------|----------|
| `DRGT_CRTT` | `0x7E40` | Tap тАФ toggle drag-scroll ┬╖ Hold тАФ toggle carret |
| `CARRETM`   | `0x7E41` | Carret while held (not placed on any board тАФ spare) |
| `CRT_TOG`   | `0x7E42` | Toggle carret |
| `CRT_SEL`   | `0x7E43` | While held: carret + Shift тАФ the ball extends the text selection (not placed тАФ spare) |
| `BTN_LOCK`  | `0x7E44` | Toggle-hold mouse button 1: drag without keeping the key down |
| `SCRB_TAB`  | `0x7E45` | Scrub: tabs |
| `SCRB_VOL`  | `0x7E46` | Scrub: volume |
| `SCRB_SPC`  | `0x7E47` | Scrub: macOS spaces |
| `SCRB_HST`  | `0x7E48` | Scrub: undo/redo |
| `SCRB_APP`  | `0x7E49` | Scrub: app switcher (holds Cmd/Alt for you) |

Typing extras: double-tap Shift starts **Caps Word** (caps until the word
ends; `CW_TOGG` also sits on RAISE next to `EE_CLR`), and **`QK_REP`** right
of it repeats the last key.

## RGB indication

| Color | State |
|-------|-------|
| ЁЯФ╡ Azure | Carret |
| ЁЯЯв Green | Drag-scroll |
| ЁЯФ┤ Red | Sniping |
| ЁЯЯа Orange | Mouse button 1 locked (`BTN_LOCK`) |
| тЪк White | Caps Word active |
| ЁЯЯб Gold | LOWER layer held |
| ЁЯЯг Purple | RAISE layer held |
| Default | Nothing active |

The list is also the priority order: trackball modes win over the lock, the
lock over Caps Word, Caps Word over layers. LEDs turn off after 5 idle
minutes (`RGB_MATRIX_TIMEOUT`).
