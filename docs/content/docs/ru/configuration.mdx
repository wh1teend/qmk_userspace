Всё настраивается в `users/wh1teend/config.h`: раскомментировать строку,
поменять значение, затем

```sh
qmk userspace-compile
```

и прошить. `EE_CLR` не нужен — слои пересеиваются автоматически.

## Включатели фич

| Дефайн | По умолч. | Включает |
|--------|:---------:|----------|
| `POINTER_ACCEL` | ✅ | акселерацию курсора от скорости |
| `DRAGSCROLL_AXIS_LOCK` | ✅ | drag-scroll держится за доминирующую ось |
| `DRAGSCROLL_INERTIA` | ✅ | флик продолжает скролл после остановки шарика; касание шарика гасит |
| `CHARYBDIS_DRAGSCROLL_REVERSE_Y` | ✅ | натуральное (как в macOS) направление скролла |
| `AUTO_MOUSE_DEFAULT_LAYER` | ❌ | нативный авто-pointer-слой при движении шарика (плюс `POINTING_DEVICE_AUTO_MOUSE_ENABLE = yes` в `rules.mk`) |

В `rules.mk` также включены Caps Word, Repeat Key и определение ОС (аккорды
прыжка по словам и переключателя приложений сами выбирают вариант под macOS
или Ctrl/Alt). В `config.h` заданы `PERMISSIVE_HOLD` (быстрые аккорды
Shift+клавиша не промахиваются в пробел+клавишу) и `RGB_MATRIX_TIMEOUT`
(подсветка гаснет после 5 минут простоя).

## Частые подстройки

<Accordions type="multiple">
<Accordion title="Курсор дёргается">
Поднять `POINTER_ACCEL_DIV` (8 → 12…16) — кривая ускорения станет мягче.
</Accordion>
<Accordion title="Каретка слишком быстрая">
Поднять `CARRET_STEP` (55 → 80…120) — больше пробега шарика на один тап.
</Accordion>
<Accordion title="Каретка вяло повторяет">
Снизить `CARRET_TAP_INTERVAL_MS` (16 → 8) — буфер флика высыпается быстрее.
</Accordion>
<Accordion title="Не тот аккорд прыжка по словам">
Выбирается по определённой ОС; зафиксировать:
`#define CARRET_WORD_KEY(arrow) C(arrow)`.
</Accordion>
</Accordions>

## Все параметры

<Tabs items={['Курсор', 'Каретка', 'Скраб', 'Drag-scroll', 'VIA']}>
<Tab value="Курсор">

При снайпе акселерация выключена.

| Дефайн | По умолч. | Смысл |
|--------|:---------:|-------|
| `POINTER_ACCEL_THRESHOLD` | 4 | тиков за отчёт, с которых начинается ускорение; ниже — движение 1:1 |
| `POINTER_ACCEL_DIV` | 8 | тиков сверх порога на +1×; меньше = агрессивнее кривая |
| `POINTER_ACCEL_LIMIT` | 4 | максимальный множитель |

</Tab>
<Tab value="Каретка">

| Дефайн | По умолч. | Смысл |
|--------|:---------:|-------|
| `CARRET_STEP` | 55 | пробег шарика на один тап стрелки; меньше = чувствительнее |
| `CARRET_STEP_SNIPING` | `×4` | пробег на один прыжок по словам (при снайпе) |
| `CARRET_TAP_INTERVAL_MS` | 16 | минимальная пауза между тапами |
| `CARRET_IDLE_TIMEOUT_MS` | 300 | сброс буфера после покоя шарика; убирает фантомные тапы |
| `CARRET_ACCEL_DIV` | 12 | скорость шарика уменьшает шаг, до ¼; больше = слабее, 0 = выкл |
| `CARRET_WORD_KEY(arrow)` | по ОС | аккорд прыжка по словам; задать — зафиксировать |
| `CARRET_REVERSE_X` / `_Y` | выкл | инверсия осей каретки |

</Tab>
<Tab value="Скраб">

| Дефайн | По умолч. | Смысл |
|--------|:---------:|-------|
| `SCRUB_STEP` | 80 | пробег шарика на один тап при скрабе |
| `SCRUB_UNDO_KEY` / `SCRUB_REDO_KEY` | `G(KC_Z)` / `LSG(KC_Z)` | что тапает `SCRB_HST` (дефолты macOS) |

</Tab>
<Tab value="Drag-scroll">

| Дефайн | По умолч. | Смысл |
|--------|:---------:|-------|
| `DRAGSCROLL_AXIS_LOCK_TIMEOUT_MS` | 250 | ось держится, пока шарик не отдохнёт столько мс |
| `DRAGSCROLL_INERTIA_ENGAGE_MS` | 20 | инерция включается, только если реальные тики шли чаще |
| `DRAGSCROLL_INERTIA_DECAY` | 15 | % роста интервала на каждый тик доката; больше = короче докат |
| `DRAGSCROLL_INERTIA_CUTOFF_MS` | 120 | докат гаснет, когда тики реже этого |

</Tab>
<Tab value="VIA">

| Дефайн | По умолч. | Смысл |
|--------|:---------:|-------|
| `DYNAMIC_KEYMAP_LAYER_COUNT` | 4 | слоёв в EEPROM; не меньше числа слоёв в `common.h` |

</Tab>
</Tabs>
