# wh1teend userspace

[English version](README.md)

Общая конфигурация для всех Charybdis (3x5 / 3x6 / 4x6): слои и кейкоды в
`common.h`, поведение в `common.c`, настройки в `config.h`. В `keymap.c`
каждой платы — только её сетка клавиш.

## Режимы трекбола

Цвет подсветки всегда показывает активный режим.

| Режим | Шарик… | Вкл/выкл | Подсветка |
|-------|--------|----------|-----------|
| **Курсор** | двигает указатель мыши, ускоряясь со скоростью | всегда включён | обычная |
| **Снайп** | двигает указатель медленно и точно, без ускорения | держать `SNIPING` · переключить `SNP_TOG` · сам, пока зажат `PT_Z`/`PT_SLSH` | 🔴 красный |
| **Drag-scroll** | скроллит, держась за одну ось | держать `DRGSCRL` · переключить `DRG_TOG` · тап `DRGT_CRTT` | 🟢 зелёный |
| **Каретка** | двигает текстовый курсор стрелками; при снайпе прыгает по словам | держать `CARRETM` · переключить `CRT_TOG` · холд `DRGT_CRTT` | 🔵 лазурный |

Drag-scroll и каретка взаимоисключающие — включение одного выключает другой.

## Pointer-слой

Держится через `PT_Z` (Z) или `PT_SLSH` (/). Ядро 3x5, одинаковое на всех
платах:

```text
QK_BOOT  DPI_RMOD DPI_MOD  S_D_RMOD S_D_MOD  │  S_D_MOD  S_D_RMOD DPI_MOD  DPI_RMOD QK_BOOT
LGUI     LALT     LCTL     LSFT     DRGT_CRTT│  CRT_TOG  RSFT     RCTL     RALT     RGUI
 · ·     DRGSCRL  SNIPING  DRG_TOG  SNP_TOG  │  SNP_TOG  DRG_TOG  SNIPING  DRGSCRL   · ·
                  BTN2     BTN1     BTN3     │  BTN3     BTN1
```

`DPI_MOD`/`DPI_RMOD` — DPI курсора выше/ниже, `S_D_MOD`/`S_D_RMOD` — то же
для DPI снайпа.

## Кастомные кейкоды

Объявлены в `common.h`, обрабатываются в `common.c`. Веб-конфигуратор
показывает их как неизвестные коды; назначаются через кнопку **Any** по
hex-значению. Правки раскладки из браузера живут до следующей прошивки —
слои пересеиваются из `keymap.c` при каждой новой сборке (комбо, тапдансы
Argos и настройки DPI выживают).

| Клавиша | Hex (Any) | Поведение |
|---------|-----------|-----------|
| `DRGT_CRTT` | `0x7E40` | Тап — вкл/выкл drag-scroll · Холд — вкл/выкл каретку |
| `CARRETM`   | `0x7E41` | Каретка, пока удерживается (в сетках не стоит — запасной) |
| `CRT_TOG`   | `0x7E42` | Вкл/выкл каретку |

## Настройка

Всё в [`config.h`](config.h): раскомментировать строку, поменять значение,
затем

```sh
qmk userspace-compile
```

и прошить. `EE_CLR` не нужен — слои пересеиваются автоматически.

### Включатели фич

| Дефайн | По умолч. | Включает |
|--------|:---------:|----------|
| `POINTER_ACCEL` | ✅ | акселерацию курсора от скорости |
| `DRAGSCROLL_AXIS_LOCK` | ✅ | drag-scroll держится за доминирующую ось |
| `CHARYBDIS_DRAGSCROLL_REVERSE_Y` | ✅ | натуральное (как в macOS) направление скролла |
| `CHARYBDIS_AUTO_POINTER_LAYER_TRIGGER_ENABLE` | ❌ | pointer-слой включается сам при движении шарика |

### Частые подстройки

- **Курсор дёргается** → поднять `POINTER_ACCEL_DIV` (8 → 12…16)
- **Каретка слишком быстрая** → поднять `CARRET_STEP` (55 → 80…120)
- **Каретка вяло повторяет** → снизить `CARRET_TAP_INTERVAL_MS` (16 → 8)
- **Linux/Windows** → `#define CARRET_WORD_KEY(arrow) C(arrow)`

### Все параметры

| Дефайн | По умолч. | Смысл |
|--------|:---------:|-------|
| **Акселерация курсора** (при снайпе выключена) | | |
| `POINTER_ACCEL_THRESHOLD` | 4 | тиков за отчёт, с которых начинается ускорение; ниже — движение 1:1 |
| `POINTER_ACCEL_DIV` | 8 | тиков сверх порога на +1×; меньше = агрессивнее кривая |
| `POINTER_ACCEL_LIMIT` | 4 | максимальный множитель |
| **Каретка** | | |
| `CARRET_STEP` | 55 | пробег шарика на один тап стрелки; меньше = чувствительнее |
| `CARRET_STEP_SNIPING` | `×4` | пробег на один прыжок по словам (при снайпе) |
| `CARRET_TAP_INTERVAL_MS` | 16 | минимальная пауза между тапами |
| `CARRET_IDLE_TIMEOUT_MS` | 300 | сброс буфера после покоя шарика; убирает фантомные тапы |
| `CARRET_ACCEL_DIV` | 12 | скорость шарика уменьшает шаг, до ¼; больше = слабее, 0 = выкл |
| `CARRET_WORD_KEY(arrow)` | `A(arrow)` | аккорд прыжка по словам |
| `CARRET_REVERSE_X` / `_Y` | выкл | инверсия осей каретки |
| **Drag-scroll** | | |
| `DRAGSCROLL_AXIS_LOCK_TIMEOUT_MS` | 250 | ось держится, пока шарик не отдохнёт столько мс |
| **VIA** | | |
| `DYNAMIC_KEYMAP_LAYER_COUNT` | 4 | слоёв в EEPROM; не меньше числа слоёв в `common.h` |
